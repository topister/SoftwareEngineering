## What is Frontend Development?

Frontend development involves designing and building the user interface (UI) and user experience (UX) of a web application. It’s everything users can see, touch, and interact with in their browser.

### Think of a website as a house:

HTML: The foundation and structure.

CSS: The paint, furniture, and decor that make it beautiful.

JavaScript: The electricity and plumbing that make it functional and interactive.

[UI/UX Designer Vs. Front End Developer. Which One Do you need?](https://www.wowmakers.com/blog/ui-ux-designer-vs-front-end-developer/#:~:text=The%20front%20end%20is%20responsible,to%20meet%20the%20user's%20needs.)

### Frontend vs. Backend Development

#### Frontend (Client-Side):

Focuses on how things look and feel to users.
Runs in the browser or your mobile-device.
Core technologies: HTML, CSS, JavaScript.

#### Backend (Server-Side):

Handles logic, database interactions, and server-side processes.
Runs on a server.
Technologies you’ve used: Python, Flask.

#### How They Work Together:

The frontend sends requests (e.g., "Get this user’s profile data") to the backend.
The backend processes the request and sends a response back to the frontend.
Example: When you click a “Submit” button, the frontend collects data and the backend stores it in a database.

#### Frontend Technologies

Technology | Role | Example Use

HTML | Structure of the webpage | Adding headings, paragraphs, images.

CSS | Styling and layout | Changing colors, fonts, layouts.

JavaScript | Interactivity | Animations, dropdowns, API calls.

1. What is the Internet?
   The internet is a global network that connects millions of computers, servers, and devices, enabling them to share information. Think of it as the world’s largest postal system, delivering digital "packages" (data) to "addresses" (IP addresses) using standardized rules called protocols.

Key Components of the Internet:

Servers: Computers that store and send information (like websites) upon request.

Clients: Your device (like a laptop or phone) that makes the request.

Protocols: Rules like HTTP and DNS that ensure data is sent and received correctly.

2. What is a Browser?
   A browser, like Chrome, Firefox, or Safari, is a tool you use to access the internet. It acts as a translator that converts the raw code (HTML, CSS, JavaScript) into the visual and interactive web pages you see.

How Does a Browser Work?

Request: When you type a web address (like www.example.com), the browser sends a request to a server.

Response: The server sends back the files needed to display the page.

Rendering: The browser reads the HTML and CSS and renders (displays) the webpage.

Key Browser Features:

Developer Tools: A built-in tool to inspect and debug your code.

Cross-Browser Compatibility: Browsers interpret code slightly differently, which is why consistent
coding practices are essential.

3. What is HTML and CSS, and How Do They Fit In?

When building a house, you start with the structure (framework) and then decorate it. HTML and CSS follow a similar principle.

HTML (HyperText Markup Language): Provides the structure of a webpage. It defines elements like headings, paragraphs, images, and links.
Example: <h1>Welcome to My Website</h1> creates a large heading.

CSS (Cascading Style Sheets): Adds style and design to the HTML structure, like colors, fonts, and layouts.
Example: h1 { color: blue; } makes the heading blue.

Why Do Browsers Need HTML and CSS?

Browsers only understand structured instructions written in specific "languages" like HTML and CSS. If we didn’t use these languages, a browser wouldn’t know how to organize or style the content.

4. How Browsers Process HTML and CSS
   Browsers interpret HTML and CSS in a step-by-step process:

Parse the HTML: Read and understand the structure of the page.

Apply the CSS: Style the elements defined in the HTML.

Render the Page: Display the combined result visually.

5. Why Structured Code Matters
   Structured code ensures:

Usability: Users can navigate and interact with web pages effectively.

Accessibility: Screen readers and assistive technologies can interpret the content for users with disabilities.

SEO: Search engines rank structured content higher, making it easier to find.
